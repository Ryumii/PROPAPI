## 2. チャネル別実行計画

### 2.1 SEO（最重要チャネル）

#### 目標キーワード

| カテゴリ | キーワード例 | 月間検索VolM | 競合度 | 優先度 |
|---|---|---|---|---|
| **ブランド検索** | 「ChiScan」「チスキャン」 | 0（新規） | 低 | 構築 |
| **ハザード系** | 「〇〇市 ハザードマップ」 | 10万+ | 中 | 最高 |
| **用途地域系** | 「〇〇区 用途地域 調べ方」 | 5万+ | 低 | 高 |
| **土地調査系** | 「土地 リスク 調べる」 | 1万+ | 低 | 高 |
| **API系** | 「ハザードマップ API」「土地情報 API」 | 500+ | 極低 | 中（開発者向け） |

#### プログラマティック SEO 戦略

**Phase 1（MVP）**: 東京23区 = 23ページ
```
/area/東京都/千代田区/
/area/東京都/渋谷区/
/area/東京都/新宿区/
...
```

**Phase 2**: 全国市区町村 = 約1,700ページ
```
/area/大阪府/大阪市北区/
/area/神奈川県/横浜市西区/
...
```

各ページに含める内容:
- 市区町村のハザード概要（洪水・土砂・津波リスクの統計）
- 用途地域の分布サマリー
- 「住所を入れて詳しく調べる」への誘導 CTA
- 周辺市区町村へのリンク（内部リンク構造）
- 構造化データ（JSON-LD: FAQPage, BreadcrumbList）

#### コンテンツ SEO

| コンテンツタイプ | 頻度 | 目的 |
|---|---|---|
| 市区町村別ハザード解説 | 自動生成 | ロングテール SEO |
| 「ハザードマップの見方」解説記事 | 月2本 | 情報系キーワード獲得 |
| 「用途地域とは？」基礎知識 | 月1本 | 認知拡大 |
| データ更新プレスリリース | 随時 | 鮮度シグナル |

## 6. リスクと Plan B

| リスク | 影響 | Plan B |
|---|---|---|
| SEO の順位が上がらない | 流入減 | リスティング広告でカバー（CPA 3,000円以内） |
| Product Hunt で不発 | 認知低 | Hacker News, Reddit r/japan 等に分散 |
| Enterprise セールスに時間がかかる | 収益遅延 | SMB (Growth) のボリュームで補填 |
| 技術記事のエンゲージメント低 | 開発者認知低 | GitHub Star 獲得施策（SDK の質向上）に注力 |
