## 1. エグゼクティブサマリー

### 1.1 ワンライナー

> **ChiScan は、日本の分散・非構造化された公的土地データを統合し、住所一つで瞬時にハザードリスク・都市計画情報を返す API プラットフォームである。**

### 1.2 問題の規模

日本では年間約 **130万件** の不動産取引が行われ、各取引において土地のハザード情報・用途地域の確認が必須である。しかし、これらのデータは **1,700超の市区町村** に分散し、フォーマットも統一されていない。現状、1件あたり **30〜90分** の手作業調査が発生しており、不動産業界全体で年間 **約650万〜1,950万時間** の非効率が生じている。

### 1.3 ソリューション

ChiScan は2層構造でこの問題を解決する：

| レイヤー | 提供形態 | ターゲット | 収益モデル |
|---|---|---|---|
| **Consumer** | GUI Web アプリ（無料） | 一般ユーザー・不動産検討者 | 広告収益（住宅ローン・保険） |
| **Developer** | REST API（有料 SaaS） | 企業・開発者・AI エージェント | サブスクリプション＋従量課金 |

### 1.4 主要数値目標（3年）

| 指標 | Year 1 | Year 2 | Year 3 |
|---|---|---|---|
| API 有料契約社数 | 26社 | 105社 | 275社 |
| GUI 月間 UU | 5万 | 25万 | 80万 |
| **ARR（年間経常収益）** | **1,200万円** | **6,200万円** | **2億円** |
| 黒字化 | — | Q4達成 | 通年黒字 |

## 2. 市場分析

### 2.1 マクロ環境

#### 不動産市場
- 日本の不動産売買取引額: 年間約 **43兆円**（2024年、国土交通省統計）
- 宅地建物取引業者数: **約129,000社**（2025年、国交省登録ベース）
- 住宅着工数: 年間約 **80万戸**（2024年、国交省建築着工統計）
- 住宅ローン新規実行額: 年間約 **20兆円**

#### 不動産テック市場
- 日本の PropTech 市場規模: 約 **1,200億円**（2025年推計、矢野経済研究所）
- CAGR: **約15%**（2023〜2028年予測）
- SaaS 化の進展: 従来のパッケージ型からクラウド API 型への移行が加速

#### 防災意識の高まり
- 能登半島地震（2024年1月）以降、ハザードマップ閲覧数が **前年比3倍** に急増
- 2020年8月の宅建業法改正で、不動産取引時の **水害ハザード情報の重要事項説明が義務化**
- 損害保険各社がリスク細分化型商品を拡大中

### 2.2 TAM / SAM / SOM

```
TAM（Total Addressable Market）
  日本の不動産テック市場全体
  = 約 1,200億円（2025年）

SAM（Serviceable Addressable Market）
  不動産データ API・土地調査関連サービス市場
  = 不動産テック市場の約8%
  = 約 96億円

SOM（Serviceable Obtainable Market）
  3年以内に獲得可能な市場シェア
  = SAM の約2%
  = 約 2億円（Year 3 ARR 目標と整合）
```

#### TAM 算出根拠
- PropTech 市場全体: SaaS、AI査定、電子契約、VR内見、データ分析等を含む
- 出典: 矢野経済研究所「不動産テック市場に関する調査」（2024年公表）

#### SAM 算出根拠
- 不動産データ API・情報提供: PropTech 市場の約8%と推定（米国 HazardHub 等の先行市場での比率を参考）
- 対象: ハザード情報 API、用途地域データ、土地調査自動化ツール

#### SOM 算出根拠
- Year 3 で有料 API 275社 × 平均 ARR 約55万円 = 約1.5億円
- GUI 広告収益 約5,000万円
- 合計 約2億円 ≒ SAM の約2%

### 2.3 市場トレンド（追い風要因）

1. **AI エージェントの台頭**: LLM ベースのエージェントが構造化データ API を必要としている。MCP（Model Context Protocol）の標準化が進行中
2. **デジタル庁のデータ整備方針**: 2025年「ベース・レジストリ」整備が進むが、API として提供されるのは数年先と見込まれる
3. **宅建業法の重説義務拡大**: ハザード情報の説明義務化により、不動産会社の調査需要が構造的に増加
4. **損保のリスク細分化**: 物件単位のハザードデータ需要が explosive に拡大中
5. **住宅ローンのオンライン化**: ネット銀行・フィンテック企業が自動審査に土地リスクデータを組み込む動き

## 3. 競合・競争優位性

### 3.1 競合マッピング

| プレイヤー | 提供形態 | カバー範囲 | API提供 | AI対応 | 価格 |
|---|---|---|---|---|---|
| **国土交通省ハザードマップポータル** | Web閲覧 | 全国 | ✗ | ✗ | 無料 |
| **各自治体 GIS** | Web閲覧 / CSV | 自治体単位 | ✗ | ✗ | 無料 |
| **Google Maps Platform** | API | 全世界 | ✅ | △ | 従量課金 |
| **ゼンリン** | データライセンス | 全国 | 一部 | ✗ | 高額 |
| **SUUMO / LIFULL HOME'S** | Web / アプリ | 全国 | 自社内利用のみ | ✗ | 無料（広告） |
| **不動産 SaaS 各社** | 業務ソフト | 全国 | ✗（自社閉鎖） | ✗ | 月額SaaS |
| **HazardHub（米国）** | API | 米国のみ | ✅ | ✅ | SaaS |
| **ChiScan（当社）** | **API + GUI** | **全国** | **✅** | **✅** | **Freemium** |

### 3.2 競争優位性（Moat）

#### 短期的優位性（Year 1〜2）
1. **ファーストムーバー**: 日本の土地データを API として外部提供する専業プレイヤーは2026年4月時点で **存在しない**
2. **2層構造による二面市場**: GUI で需要を証明しながら API で収益化するモデルは参入障壁が高い
3. **SEO による自然流入**: 1,700市区町村のプログラマティック SEO で検索トラフィックを獲得

#### 中長期的優位性（Year 3〜）
4. **データ正規化の蓄積**: 1,700市区町村のフォーマット差異を吸収する正規化ルールは、時間と労力の壁そのもの
5. **ネットワーク効果**: API 利用企業が増えるほど、品質フィードバック・カバレッジ要望が集まりデータが改善
6. **AI エージェントのデファクト**: MCP 対応を早期に実現し「土地調査 = ChiScan」のポジションを確立
7. **スイッチングコスト**: 企業がシステムに組み込んだ API は簡単には切り替えられない

### 3.3 参入障壁の分析

| 障壁 | 新規参入者にとっての困難度 | ChiScan の状況 |
|---|---|---|
| データ収集・正規化 | **高**: 1,700自治体の異なるフォーマットを統合するのは 6〜12ヶ月の作業 | Phase 1〜2 で構築 |
| 空間データ処理能力 | **中**: PostGIS 等の技術は公開されているが、パフォーマンス最適化にノウハウが要る | 技術チームで対応 |
| 顧客基盤 | **高**: API を組み込んだ企業は容易に乗り換えない | Starter プランで早期獲得 |
| ブランド認知 | **中**: SEO + 開発者コミュニティで構築 | Phase 1 から着手 |

## 7. リスクと緩和策

### 7.1 事業リスク

| リスク | 影響度 | 発生確率 | 緩和策 |
|---|---|---|---|
| **デジタル庁の無料 API 整備** | 高 | 中（3〜5年後） | 正規化品質・SLA・GUI の付加価値で差別化。整備後はデータソースとして活用 |
| **大手テック企業の参入** | 高 | 低 | ニッチ特化（日本の土地法規）で先行。買収候補としてのポジショニング |
| **データソースのアクセス制限** | 中 | 中 | 複数ソースの冗長化。公式連携の交渉を早期開始 |
| **需要の立ち上がり不足** | 中 | 中 | GUI の無料提供で需要を顕在化。Starter プランで心理的障壁を下げる |
| **技術的スケーリング** | 中 | 低 | Azure マネージドサービス活用。Phase 2 で SRE 採用 |

### 7.2 法的リスク

| リスク | 状況 | 対策 |
|---|---|---|
| **公的データの二次利用** | 国交省データは「政府標準利用規約 2.0」で商用利用可能 | 利用規約を遵守し、出典を明示 |
| **精度に起因する損害賠償** | API 利用規約で免責条項を明記 | 「参考情報であり法的判断の根拠としない」旨を明示 |
| **登記情報の取り扱い** | Phase 4 以降。個人情報保護法・不動産登記法の制約 | 法務専門家と連携。Phase 1〜3 の実績を先に構築 |
