## Epic 1: GUI Web アプリ — 住所検索・リスク表示

### US-101: 住所でハザード情報を検索する
> **As a** マイホーム検討中の一般ユーザー  
> **I want to** 住所を入力してハザードリスクを検索したい  
> **So that** 候補物件の安全性を簡単に確認できる

**優先度**: P0（MVP）  
**Story Point**: 8

**受入条件**:
- [ ] 住所入力フィールドに日本語住所を入力できる
- [ ] 入力途中でサジェスト候補が表示される（オートコンプリート）
- [ ] 「検索」ボタン押下後、3秒以内に結果が表示される
- [ ] 洪水・土砂災害・津波・液状化の各リスクが表示される
- [ ] 各リスクは5段階（なし・低・中・高・非常に高い）でスコア表示される
- [ ] データの出典と最終更新日が表示される
- [ ] 該当住所が見つからない場合、エラーメッセージが表示される

---

### US-102: 地図上でハザードゾーンを確認する
> **As a** マイホーム検討中の一般ユーザー  
> **I want to** 地図上でハザードゾーンの範囲を視覚的に確認したい  
> **So that** 周辺エリアのリスクも含めて判断できる

**優先度**: P0（MVP）  
**Story Point**: 8

**受入条件**:
- [ ] 検索結果に地図が表示される
- [ ] 検索した地点にピンが立つ
- [ ] 洪水浸水想定区域がポリゴンでオーバーレイ表示される
- [ ] 土砂災害警戒区域がポリゴンでオーバーレイ表示される
- [ ] 各ハザードの凡例が表示される
- [ ] ズーム・パンで周辺を探索できる
- [ ] レイヤーの ON/OFF を切り替えられる

---

### US-103: 用途地域・建築規制を確認する
> **As a** マイホーム検討中の一般ユーザー  
> **I want to** 用途地域・建ぺい率・容積率を確認したい  
> **So that** どんな建物が建てられるか把握できる

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] 用途地域の名称（例:「第一種低層住居専用地域」）が表示される
- [ ] 建ぺい率と容積率が数値で表示される
- [ ] 防火指定（防火地域/準防火地域/指定なし）が表示される
- [ ] 用途地域の簡単な説明文が表示される
- [ ] 用途地域が地図上で色分け表示される

---

### US-104: リスクサマリーを確認する
> **As a** 不動産知識のない一般ユーザー  
> **I want to** 総合的なリスクスコアをわかりやすく見たい  
> **So that** 専門知識なしにその土地のリスクを把握できる

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] 総合リスクスコアが5段階で表示される（★表示）
- [ ] スコアの色分け（緑〜赤）が視覚的にわかりやすい
- [ ] 各ハザード項目の個別スコアも表示される
- [ ] 「このスコアの意味」を解説するツールチップがある
- [ ] スコア算出のロジック（どのハザードを重視しているか）の説明リンクがある

---

### US-105: 検索結果を共有する
> **As a** マイホーム検討中のユーザー  
> **I want to** 検索結果を家族や不動産エージェントに共有したい

---

### US-106: 複数住所を比較する
> **As a** 物件を比較検討中のユーザー

## Epic 2: REST API — ハザード情報

### US-201: API Key を発行する
> **As a** 不動産テック CTO  
> **I want to** API Key を発行してすぐに API を試したい  
> **So that** サービスへの組み込み可否を即座に検証できる

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] メールアドレスで開発者アカウントを作成できる
- [ ] アカウント作成後、即座に API Key が発行される
- [ ] Starter プラン（無料・1,000件/月）が自動適用される
- [ ] ダッシュボードで API Key の確認・再発行ができる
- [ ] API Key はヘッダー `X-API-Key` で送信する

---

### US-202: 住所からハザード情報を取得する
> **As a** 不動産テック CTO  
> **I want to** 住所を API に POST してハザードリスクを JSON で受け取りたい  
> **So that** 自社の物件ページにリスク情報を表示できる

**優先度**: P0（MVP）  
**Story Point**: 13

**受入条件**:
- [ ] `POST /v1/land/inspect` に住所を送信できる
- [ ] レスポンスに洪水・土砂・津波・液状化のリスクレベルが含まれる
- [ ] レスポンスに用途地域・建ぺい率・容積率が含まれる
- [ ] レスポンスタイム P95 < 500ms
- [ ] 正規化された住所が `address_normalized` フィールドに返る
- [ ] データの出典と更新日が `meta` フィールドに含まれる
- [ ] 信頼度スコア（`confidence`）が含まれる
- [ ] 該当データなしの場合、適切なエラーレスポンス（404）が返る

---

### US-203: 緯度経度からハザード情報を取得する
> **As a** 地図ベースのアプリを開発する開発者  
> **I want to** 緯度経度から直接ハザード情報を取得したい  
> **So that** 地図上のクリック座標から即座にリスクを表示できる

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] `POST /v1/land/inspect` に `lat` / `lng` のみで送信できる
- [ ] `address` フィールドなしでも緯度経度から結果が返る
- [ ] 住所と緯度経度の両方が指定された場合、緯度経度を優先する
- [ ] 日本国外の座標が指定された場合、400 エラーが返る

---

### US-204: ハザード情報のみを取得する
> **As a** 保険会社のデータエンジニア  
> **I want to** ハザード情報のみを軽量に取得したい  
> **So that** 大量の引受審査を高速に処理できる

**優先度**: P0（MVP）  
**Story Point**: 3

**受入条件**:
- [ ] `GET /v1/hazard?address=...` でハザード情報のみ取得できる
- [ ] `GET /v1/hazard?lat=...&lng=...` でも取得できる
- [ ] レスポンスに `zoning` フィールドは含まれない
- [ ] `/v1/land/inspect` より高速（P95 < 300ms）

---

### US-205: 用途地域情報のみを取得する
> **As a** 建築設計事務所のエンジニア  
> **I want to** 用途地域・建ぺい率・容積率のみを取得したい  
> **So that** 設計の初期検討に必要なデータだけ高速に取得できる

**優先度**: P0（MVP）  
**Story Point**: 3

**受入条件**:
- [ ] `GET /v1/zoning?address=...` で用途地域情報のみ取得できる
- [ ] レスポンスに `hazard` フィールドは含まれない
- [ ] 用途地域・建ぺい率・容積率・防火指定が含まれる

---

### US-206: バッチ処理で大量の住所を一括処理する
> **As a** 損保会社のデータエンジニア  
> **I want to** 数千件の住所を一括で処理したい  
> **So that** ポートフォリオ全体のリスク評価を効率的に行える

**優先度**: P1（Phase 2）  
**Story Point**: 13

**受入条件**:
- [ ] `POST /v1/batch` に最大 1,000 件の住所を送信できる
- [ ] レスポンスは各住所ごとの結果を配列で返す
- [ ] 一部の住所が失敗しても、成功分は結果を返す
- [ ] 処理状況を `GET /v1/batch/{job_id}` で確認できる
- [ ] 完了時にWebhook通知を受け取れる（オプション）
- [ ] Business プラン以上で利用可能

---

### US-207: データ更新履歴を確認する
> **As a** コンプライアンス重視の金融機関  
> **I want to** 特定住所のデータ更新履歴を確認したい  
> **So that** 監査時にデータの鮮度と変更を追跡できる

**優先度**: P1（Phase 2）  
**Story Point**: 5

**受入条件**:
- [ ] `GET /v1/change-log?address=...` で更新履歴を取得できる
- [ ] 各更新のタイムスタンプと変更内容が含まれる
- [ ] Enterprise プラン以上で利用可能

---

### US-302: 利用量を確認する
> **As a** API を利用中の開発者  
> **I want to** 今月のリクエスト数と残量を確認したい  
> **So that** 上限に達する前にプランをアップグレードできる

**優先度**: P0（MVP）  
**Story Point**: 3

**受入条件**:

- [ ] レスポンスヘッダーに残量情報（`X-RateLimit-Remaining`）が含まれる

---

### US-303: プランをアップグレードする

**受入条件**:
- [ ] ダッシュボードからプラン変更画面にアクセスできる

## Epic 4: 認証・セキュリティ

### US-401: API Key 認証でリクエストする
> **As a** API 利用者  
> **I want to** API Key で安全にリクエストしたい  
> **So that** 自分のアカウントでのみリクエストが計上される

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] `X-API-Key` ヘッダーなしのリクエストは 401 を返す
- [ ] 無効な API Key は 403 を返す
- [ ] 有効な API Key でのみリクエストが成功する
- [ ] API Key はダッシュボードで無効化・再発行できる
- [ ] API Key はハッシュ化して保存される

---

### US-402: レート制限を適用する
> **As a** ChiScan 運営者  
> **I want to** プランに応じたレート制限を適用したい  
> **So that** サービスの安定性を保ちつつ公平に利用できる

**優先度**: P0（MVP）  
**Story Point**: 5

**受入条件**:
- [ ] 各プランの月間上限を超えたリクエストは 429 を返す
- [ ] レスポンスに `Retry-After` ヘッダーが含まれる
- [ ] 秒間レート制限（Starter: 10req/s, Growth: 50req/s, Business: 200req/s）が適用される
- [ ] Enterprise は個別のレート設定が可能

## Epic 7: データ管理（内部）

### US-701: ハザードデータを定期更新する
> **As a** ChiScan のデータエンジニア  
> **I want to** 国交省の最新ハザードデータを自動取得・反映したい  
> **So that** ユーザーに常に最新のリスク情報を提供できる

**優先度**: P0（MVP）  
**Story Point**: 13

**受入条件**:
- [ ] Airflow DAG で定期スケジュール（月次）が設定されている
- [ ] データ取得→正規化→品質チェック→PostGIS 反映の一連のパイプラインが動作する
- [ ] 品質チェックで異常値が検出された場合、アラートが送信される
- [ ] 更新完了後、`meta.data_updated_at` が更新される
- [ ] 更新前後のデータ差分がログに記録される

---

### US-702: 住所を正規化する
> **As a** API 利用者  
> **I want to** 表記ゆれのある住所でも正しく検索結果が返ってほしい  
> **So that** ユーザー入力の品質に依存せずサービスを利用できる

**優先度**: P0（MVP）  
**Story Point**: 8

**受入条件**:
- [ ] 「東京都渋谷区渋谷2-24-12」と「渋谷区渋谷二丁目24番12号」で同じ結果が返る
- [ ] 大字・字の省略に対応する
- [ ] 枝番・号・方書きを正しくパースする
- [ ] マッチ失敗時は `confidence` を低く設定して近似結果を返す
- [ ] 正規化後の住所が `address_normalized` に返る
